# Findings

## Core Purpose
- **Target**: API and Web application test cases.
- **Types**: Functional and non-functional.
- **Output**: Jira format (readable description, steps, expected results, priority, etc.).

## Tech Stack
- **Backend**: Node.js with TypeScript (Express.js recommended).
- **Frontend**: React (TypeScript) with Vite.
- **Styling**: Vanilla CSS or Tailwind (based on prompt I will use premium Vanilla CSS with modern aesthetics).
- **LLM Support**: Ollama, LM Studio, Groq, OpenAI, Claude, Gemini.

## Research & Design
- **UI Architecture**:
    - Sidebar: Requirement history management.
    - Main Panel: Input for requirements and display area for generated test cases.
    - Settings: Configuration for multiple LLM providers (Endpoints, API Keys).
    - Features: "Test Connection" for provider validation, "Save" for configuration persistence.

## Constraints
- Protocol 0: No code/scripts until Blueprint is approved.
- Output MUST be in Jira format.
- Multi-provider support is mandatory.
